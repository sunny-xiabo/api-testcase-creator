# API 测试评审期望索引

> 管理规范：
> - 编号格式：`[维度缩写-序号]`，序号两位补零
> - 新增时追加到对应分类末尾，禁止修改已有编号
> - 废弃的评审点在描述后标注 `[已废弃]`，不直接删除

---

## 参数校验（PARAM）

- [PARAM-01] 所有必填参数均有缺失测试用例
- [PARAM-02] 所有字段均有类型错误测试用例
- [PARAM-03] 有数值约束的字段（min/max）均有边界值测试
- [PARAM-04] 有长度约束的字段（minLength/maxLength）均有边界值测试
- [PARAM-05] 有枚举值的字段均有非法值测试
- [PARAM-06] 有正则约束的字段（pattern）均有不匹配测试

---

## 鉴权测试（AUTH）

- [AUTH-01] 所有需认证接口均有无 Token 测试
- [AUTH-02] 所有需认证接口均有伪造 Token 测试
- [AUTH-03] 权限分级接口有越权测试（低权限访问高权限资源）

---

## 业务场景（BIZ）

- [BIZ-01] 核心业务流程有完整正向用例
- [BIZ-02] 业务规则约束有反向用例（如：已关闭的订单不能支付）
- [BIZ-03] 业务状态流转有完整覆盖

---

## 数据校验（DATA）

- [DATA-01] 成功响应有状态码断言
- [DATA-02] 关键响应字段有值断言（不只是状态码）
- [DATA-03] 错误响应有错误信息断言（不只是状态码）
- [DATA-04] 列表响应有数量/分页断言

---

## 流程覆盖（CHAIN）

- [CHAIN-01] 上下游接口有依赖关系测试
- [CHAIN-02] 数据传递有正确性验证（如创建的 ID 用于后续查询）
- [CHAIN-03] 异常链路有降级行为测试

---

## 性能测试（PERF）

- [PERF-01] 关键接口有响应时间断言
- [PERF-02] 大数据量场景有考虑

---

## 安全测试（SEC）

- [SEC-01] 用户输入字段有注入测试
- [SEC-02] 敏感数据有暴露检查

---

## 幂等并发（IDEM）

- [IDEM-01] 写接口有幂等性测试
- [IDEM-02] 并发操作同一资源有测试
