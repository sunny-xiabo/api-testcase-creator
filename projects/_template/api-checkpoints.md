# API 测试检查点索引

> 管理规范：
> - 编号格式：`[域缩写-序号]`，序号两位补零（如 AUTH-01）
> - 新增时追加到对应分类末尾，禁止修改已有编号
> - 废弃的检查点在描述后标注 `[已废弃]`，不直接删除
> - 通用模式（必填缺失、类型错误、边界值）由程序自动生成，无需在此维护

---

## API-AUTH: 鉴权与权限

- [AUTH-001] 无 Token 访问需认证接口 → 401
- [AUTH-002] Token 过期后访问 → 401，错误信息包含"expired"
- [AUTH-003] 伪造/篡改 Token → 401
- [AUTH-004] 低权限用户访问高权限接口 → 403
- [AUTH-005] [项目特定] 多租户隔离：A 租户不能访问 B 租户数据

---

## API-BIZ: 业务逻辑

- [BIZ-001] [项目特定] 创建订单后库存应减少
- [BIZ-002] [项目特定] 已支付订单不能重复支付
- [BIZ-003] [项目特定] 取消订单后状态变为 cancelled

---

## API-DATA: 数据校验

- [DATA-001] 响应 JSON 结构与接口文档一致
- [DATA-002] 列表接口返回条数 <= 请求的 page_size
- [DATA-003] 时间字段格式符合 ISO 8601 或项目约定
- [DATA-004] [项目特定] 金额字段精度：分/元、小数位数
- [DATA-005] 分页接口的 total 字段与实际数据量一致

---

## API-IDEM: 幂等性与并发

- [IDEM-001] 同一请求短时间内重复提交 → 返回幂等结果（非报错）
- [IDEM-002] [项目特定] 支付回调重复通知 → 只处理一次
- [IDEM-003] 并发修改同一资源 → 最终状态一致
- [IDEM-004] 并发创建相同唯一约束的资源 → 只有一个成功

---

## API-PERF: 性能

- [PERF-001] 单接口响应时间 < 500ms
- [PERF-002] [项目特定] 列表接口在 N 条数据下响应时间 < 2s
- [PERF-003] 批量接口单次上限校验（如最多 100 条）

---

## API-SEC: 安全

- [SEC-001] SQL 注入测试（参数中包含特殊字符 ' " ; --）
- [SEC-002] XSS 测试（响应中未转义的用户输入）
- [SEC-003] [项目特定] 敏感信息不出现在 URL 参数中
- [SEC-004] 路径遍历测试（../../etc/passwd）

---

## API-CHAIN: 跨接口流程

- [CHAIN-001] [项目特定] 登录 → 创建 → 支付 → 查询 全链路
- [CHAIN-002] [项目特定] 上游接口失败时下游接口的降级行为
- [CHAIN-003] 删除资源后，关联查询应返回 404 或空
