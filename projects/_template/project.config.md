## 业务域列表

> 生成用例时，将优先从以下业务域中匹配接口分组。

| 域标识 | 中文名称 | 说明 |
|--------|----------|------|
| [USER] | [用户模块] | [登录、注册、用户信息] |
| [ORDER] | [订单模块] | [创建、查询、支付、取消] |
| [PRODUCT] | [商品模块] | [商品列表、详情、搜索] |

> [!NOTE]
> 新增业务域时，同步在 api-checkpoints.md 中新增对应分类。

## 默认行为配置

| 配置项 | 当前值 | 说明 |
|--------|--------|------|
| 用例ID前缀 | [TC] | 生成用例时的编号前缀 |
| 默认优先级规则 | P0=核心正向+鉴权异常，P1=参数校验+边界值，P2=幂等并发，P3=性能 | LLM 分配优先级的参考 |
| 评审默认应用维度 | PARAM-01,PARAM-02,AUTH-01,DATA-01 | 每次评审时默认勾选的评审点 |
| 历史文件是否 Git 追踪 | 否（已加入 .gitignore） | 改为「是」则从 .gitignore 中删除对应规则 |
