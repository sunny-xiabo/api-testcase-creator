# API 测试运行 Skill — api-testcase-runner

> 运行由 api-testcase-creator 生成的 pytest 测试代码，支持多种运行模式和报告格式。
> 本 Skill 负责运行测试，生成测试用例请使用 `/api-testcase-creator`。

---

## 0. 初始化检查（每次触发自动执行）

1. 检查 `.api-testcase-assets/` 目录是否存在：
   - 存在 → 继续
   - 不存在 → 提示用户先运行 `/api-testcase-creator` 生成测试用例，并**中止流程**。

2. 检查 `.api-testcase-assets/scripts/run_tests.py` 是否存在：
   - 存在 → 继续
   - 不存在 → 提示用户重新部署框架，并**中止流程**。

3. 读取 `.api-testcase-assets/project.config.md`（若存在），从中载入以下上下文：
   - 项目名称
   - API 基础地址
   - 认证方式

4. 初始化通过后，输出：

```
[OK] 框架加载成功
[DIR] 运行脚本：.api-testcase-assets/scripts/run_tests.py
[DIR] 项目配置：.api-testcase-assets/project.config.md（已载入上下文）
>> 开始 API 测试运行流程，共 5 个阶段。
```

## 5. 失败分析（阶段 5/5）

**触发条件**：存在失败用例且用户选择分析

**执行步骤：**

### 5.1 失败详情分析

对每个失败用例进行分析：

```markdown
## 失败分析

### TC-005: 重复提交订单

**基本信息**
- 接口：POST /api/orders
- 测试点：同一请求重复提交应返回幂等结果
- 优先级：P0

**请求详情**
```json
{
  "method": "POST",
  "path": "/api/orders",
  "body": {
    "product_id": "P001",
    "quantity": 2
  }
}
```

**预期结果**
- 状态码：200
- 返回已有 order_id（幂等）

**实际结果**
- 状态码：400
- 响应体：`{"error": "order already exists"}`

**分析**
- 接口未实现幂等逻辑
- 重复提交时返回 400 而非返回已有资源
- 建议与后端确认幂等策略

**建议**
- [ ] 确认接口是否应支持幂等
- [ ] 如果不支持，修改用例预期为 400
- [ ] 如果支持，提交 bug 给后端
```

### 5.2 失败分类汇总

```markdown
### 失败分类汇总

| 失败类型 | 数量 | 说明 |
|----------|------|------|
| 断言失败 | 2 | 预期结果与实际不符 |
| 状态码错误 | 1 | 返回的状态码不符合预期 |
| 超时 | 1 | 请求超时 |

### 可能原因

1. **断言失败**：接口行为与文档不一致，或测试数据问题
2. **状态码错误**：接口实现变更，或参数处理逻辑不同
3. **超时**：网络问题，或接口性能问题
```

## 附录：运行模式说明

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 冒烟模式 | 只运行 P0 用例 | 首次运行、快速验证、CI/CD 快速检查 |
| 完整模式 | 运行全部用例 | 发版前全面验证、定期回归测试 |
| 干跑模式 | 只收集不运行 | 验证代码正确性、检查用例数量 |

## 附录：报告格式说明

| 格式 | 说明 | 适用场景 |
|------|------|----------|
| HTML | 交互式报告，含图表 | 团队分享、详细分析 |
| Markdown | 轻量文本报告 | 归档、Git 提交、快速查看 |
| JUnit XML | 标准 CI 格式 | Jenkins、GitLab CI 等集成 |

## 附录：常见问题处理

### 问题 1：config.yaml 未配置

```
[ERROR] config.yaml 中的 base_url 未配置

解决方案：
1. 编辑 <tests目录>/config.yaml
2. 填写实际的 API 地址，如：https://api.example.com
3. 重新运行测试
```

### 问题 2：依赖未安装

```
[ERROR] 缺少依赖：pytest

解决方案：
pip install pytest requests PyYAML XTestRunner

或使用 requirements.txt：
pip install -r <tests目录>/requirements.txt
```

### 问题 3：API 不可达

```
[ERROR] 无法连接到 API 服务：<base_url>

解决方案：
1. 检查 API 服务是否正常运行
2. 检查网络连接
3. 检查防火墙设置
4. 确认 base_url 配置正确
```

### 问题 4：用例收集失败

```
[ERROR] pytest 用例收集失败

可能原因：
1. 测试代码语法错误
2. conftest.py 中的 fixture 定义有误
3. import 路径问题

解决方案：
1. 运行 python3 -m py_compile <file.py> 检查语法
2. 检查 conftest.py 中的 fixture 定义
3. 确保 api_client.py 在正确的路径
```
